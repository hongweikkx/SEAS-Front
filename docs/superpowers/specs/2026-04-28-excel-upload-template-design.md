# 成绩 Excel 上传模板与自适应识别设计文档

## 1. 背景与目标

SEAS 系统的「新建分析」页面需要支持用户上传成绩 Excel 文件，系统根据上传内容自动识别格式并生成相应的分析视图。

**核心目标**：
- 降低首次使用门槛：用户不需要了解系统内部的数据结构，传什么我们认什么
- 支持渐进式深度：先传总分可做大维度分析，补充题目明细后可下钻到题目级别
- 一次上传即可：支持多 Sheet 的完整成绩簿，也支持单 Sheet 的简易成绩表

## 2. 模板格式规范

### 2.1 简单模式（单 Sheet）

**适用场景**：用户只有各科总分，想快速生成学科汇总、班级汇总、四率分析。

**结构**：
- 1 个 Sheet，建议命名为 `总成绩`（非强制）
- 列：姓名 | 班级 | {学科1} | {学科2} | ... | {学科N}
- 行：每个学生一行

**示例**：

| 姓名 | 班级 | 语文 | 数学 | 英语 |
|------|------|------|------|------|
| 张三 | 一班 | 85 | 92 | 78 |
| 李四 | 二班 | 90 | 88 | 95 |

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | string | ✅ | 学生姓名 |
| 班级 | string | ✅ | 班级名称，如"一班"、"七年级(1)班" |
| {学科名} | number | 至少一科 | 列名即学科名，值为该科总分 |

### 2.2 完整模式（多 Sheet）

**适用场景**：用户有各科答题明细，想做题目级别的下钻分析（每题得分分布、错题分析等）。

**结构**：
- Sheet 1「总成绩」：与简单模式结构完全一致
- Sheet 2+「{学科名}」：每个科目一个 Sheet，Sheet 名须与总成绩表中的学科列名匹配

**科目明细 Sheet 结构**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 姓名 | string | ✅ | 须与总成绩 Sheet 中的姓名一致（用于关联） |
| 班级 | string | ✅ | 班级名称 |
| 题{n} | number | 至少一题 | 第 n 题得分，n 从 1 开始递增 |
| 总分 | number | ✅ | 该科目总分，用于与总成绩表交叉校验 |

**示例（Sheet「语文」）**：

| 姓名 | 班级 | 题1 | 题2 | 题3 | 总分 |
|------|------|-----|-----|-----|------|
| 张三 | 一班 | 5 | 8 | 12 | 85 |
| 李四 | 二班 | 7 | 6 | 10 | 88 |

### 2.3 Sheet 命名与匹配规则

1. 系统优先查找名为 `总成绩` 的 Sheet 作为总分表
2. 若无 `总成绩`，取**第一个 Sheet** 作为总分表
3. 其余 Sheet 视为科目明细，Sheet 名即学科名
4. 科目明细 Sheet 的学科名须与总成绩表中的学科列名**匹配**（支持模糊匹配，如"语文"匹配"语 文"）
5. 未匹配的科目明细 Sheet 将被忽略，并提示用户

## 3. 自适应识别逻辑

### 3.1 识别流程

```
用户上传 Excel
    │
    ▼
读取所有 Sheet 名称
    │
    ▼
确定「总成绩」Sheet（优先找「总成绩」，否则取第一个）
    │
    ▼
解析总成绩 Sheet 的表头
    │
    ├── 识别出「姓名」列
    ├── 识别出「班级」列
    └── 识别出学科列（排除姓名、班级后的数值列）
    │
    ▼
检查是否有其他 Sheet
    │
    ├── 无其他 Sheet ──→ 标记为「简单模式」
    │
    └── 有其他 Sheet ──→ 逐个匹配学科名
            │
            ├── 匹配成功 ──→ 标记为「完整模式」，该科目支持下钻
            └── 匹配失败 ──→ 忽略该 Sheet，记录警告
```

### 3.2 识别结果展示

上传成功后，系统展示识别摘要：

> ✅ 已识别 **3 门学科**：语文、数学、英语  
> 👥 共 **86 名学生**，来自 **3 个班级**  
> 📊 模式：**简单模式**（如需题目级分析，可后续补充上传科目明细）

或：

> ✅ 已识别 **3 门学科**：语文、数学、英语  
> 👥 共 **86 名学生**，来自 **3 个班级**  
> 📊 模式：**完整模式**（语文、数学支持题目级下钻，英语暂无明细）

## 4. 前端 UI 设计

### 4.1 上传页面布局

参考 mockup v4（已确认）：

- **主上传区**：大文件拖拽区域，底部三个彩色标签突出卖点

## 5. 模板下载实现

采用**前端动态生成**方案（方案 B）。

### 5.1 依赖

```bash
npm install xlsx
```

### 5.2 实现位置

`src/lib/templates.ts` 或 `src/components/create/template-download.ts`

### 5.3 下载函数

```typescript
import * as XLSX from 'xlsx';

export function downloadSimpleTemplate() {
  const data = [
    ['姓名', '班级', '语文', '数学', '英语'],
    ['张三', '一班', 85, 92, 78],
    ['李四', '二班', 90, 88, 95],
  ];
  const ws = XLSX.utils.aoa_to_sheet(data);
  const wb = XLSX.utils.book_new();
  XLSX.utils.book_append_sheet(wb, ws, '总成绩');
  XLSX.writeFile(wb, '成绩表.xlsx');
}

export function downloadFullTemplate() {
  const wb = XLSX.utils.book_new();

  // Sheet 1: 总成绩
  const summaryData = [
    ['姓名', '班级', '语文', '数学', '英语'],
    ['张三', '一班', 85, 92, 78],
    ['李四', '二班', 90, 88, 95],
  ];
  const summaryWs = XLSX.utils.aoa_to_sheet(summaryData);
  XLSX.utils.book_append_sheet(wb, summaryWs, '总成绩');

  // Sheet 2: 语文
  const chineseData = [
    ['姓名', '班级', '题1', '题2', '题3', '总分'],
    ['张三', '一班', 5, 8, 12, 85],
    ['李四', '二班', 7, 6, 10, 88],
  ];
  const chineseWs = XLSX.utils.aoa_to_sheet(chineseData);
  XLSX.utils.book_append_sheet(wb, chineseWs, '语文');

  // Sheet 3: 数学
  const mathData = [
    ['姓名', '班级', '题1', '题2', '题3', '总分'],
    ['张三', '一班', 6, 9, 15, 92],
    ['李四', '二班', 7, 8, 14, 88],
  ];
  const mathWs = XLSX.utils.aoa_to_sheet(mathData);
  XLSX.utils.book_append_sheet(wb, mathWs, '数学');

  XLSX.writeFile(wb, '成绩表.xlsx');
}
```

### 5.4 按钮集成

在「简单模式」和「完整模式」卡片底部各放一个下载按钮：

- 「📥 下载简单模式模板」→ 调用 `downloadSimpleTemplate()`
- 「📥 下载完整模式模板」→ 调用 `downloadFullTemplate()`

## 6. 错误处理与边界情况

### 6.1 上传阶段

| 错误场景 | 处理方式 |
|---------|---------|
| 文件格式非 .xlsx | 提示「请上传 Excel (.xlsx) 格式文件」 |
| 文件超过 20MB | 提示「文件过大，请压缩后重新上传」 |
| 找不到「姓名」列 | 提示「未识别到「姓名」列，请检查表头」 |
| 找不到「班级」列 | 提示「未识别到「班级」列，请检查表头」 |
| 找不到任何学科列 | 提示「未识别到学科成绩列，请检查表头格式」 |
| 科目明细 Sheet 名与总成绩表不匹配 | 提示「Sheet「xxx」未匹配到对应学科，已忽略」 |
| 科目明细中学生姓名与总成绩表不一致 | 仅匹配成功的学生参与题目分析，不匹配的跳过并记录 |

### 6.2 数据校验

- 数值列中出现非数字：跳过该行并记录警告
- 总分校验：科目明细中的「总分」列可与总成绩表中的对应学科总分做交叉校验，偏差过大时提示用户确认

## 7. 后续补充上传机制

### 7.1 触发场景

在单科分析页面（如「语文」分析），如果该科目没有题目明细数据，在下钻入口显示：

> "上传 [语文] 答题明细，即可查看每题得分分布"

### 7.2 上传限制

- 只接受单个 Sheet 的 Excel
- Sheet 中须包含「姓名」列用于与学生关联
- 上传后立即刷新该科目的题目级分析视图

### 7.3 数据合并

- 补充上传的数据与原有考试数据合并存储
- 同一学生的同一科目重复上传时，以最新上传为准
